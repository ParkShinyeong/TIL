##model과 migration의 차이!

**models**

- 앱으로 **데이터베이스 테이블로 데이터(테이블 행)을 보낼 때마다 사용**된다.
- 유효한 데이터베이스 항목으로 간주되는 항목을 결정한다. sql 객체 관계형관리자에서 각 후보 테이블 행은 모델을 통과하며, 행에 필요한 열이 있는지 확인한다.모두 확인하면, 데이터베이스에 입력된다.

##

**migration**

- 테이블의 열을 설정하거나 행을 대량으로 추가하는데 사용된다.
- 모델이나 테이블에 **변경 사항으로 CLI에서 사용**한다. 데이터베이스에서 변경사항에 대한 커밋이나 로그와 같이 처리한다.
- up, down 함수가 있는 데이터베이스 작업의 집합
  - **up** : 데이터베이스를 변경
  - **down:** up 기능이 실행되기 전의 상태로 데이터베이스를 복원하려고 시도
- 데이터베이스의 변경 사항을 추적하는데 유용하고, 데이터베이스 환경을 신속하게구성하고 복제할 수 있는 훌륭한 도구이다.

#

따라서 동일한 데이터베이스 테이블에 대한 모델과 마이그레이션은 반복적인 것 처럼보일 수 있지만, **모델은 데이터 베이스에 들어가는 데이터를 제어**하고, **마이그래이션 자체가 데이터베이스를 실제로 변경하는 것**이다!
